Key Insights

1

Efficiency Paradox

The 0.8B model is 545x more efficient than the 9B model (272.5 vs 0.5 t/s/B). This is due to the MoE (Mixture-of-Experts) architecture overhead in larger models.

2

Latency Scaling

Time to first token increases 11x from 0.8B (0.6s) to 9B (6.7s). This impacts real-time applications significantly.

3

Speed Variability

The 0.8B model shows higher variability (85-312 t/s) depending on task complexity, while larger models are more consistent but significantly slower.

4

Architecture Trade-offs

Larger models use MoE architecture for better reasoning but sacrifice raw throughput. The 0.8B uses dense architecture optimized for speed.
